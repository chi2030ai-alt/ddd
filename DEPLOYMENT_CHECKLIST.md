## ⚠️ 已知限制和注意事项

### 当前实现的限制
```
1. 数据存储在 React State
   - 刷新页面后重置为初始数据
   - 生产环境需接入真实数据库

2. AI 功能依赖 Gemini API
   - 需要有效的 API 密钥
   - API 请求有频率限制
   - 网络连接中断会导致功能失败

3. 没有用户认证
   - 任何访问 URL 的人都可以使用
   - 生产环境应添加认证层

4. 没有权限管理
   - 所有用户都有完整权限
   - 生产环境应实现 RBAC

5. 没有审计日志
   - 没有操作记录追踪
   - 生产环境应记录重要操作
```

### 推荐的后续工作
```
高优先级:
[ ] 接入真实数据库 (PostgreSQL/MySQL)
[ ] 实现用户认证系统
```

## 📞 支持和故障排除

### 常见问题

**Q: 应用启动时报错 "GEMINI_API_KEY 未定义"**
A: 检查 `.env.local` 文件，确保设置了 GEMINI_API_KEY

**Q: 数据在页面刷新后消失**
A: 这是预期行为，因为数据存储在 React State。需要接入数据库持久化。

**Q: AI 功能无响应**
A: 检查网络连接，验证 API 密钥有效性，查看浏览器控制台错误

**Q: 页面加载缓慢**
A: 检查浏览器网络标签，优化 Gemini API 调用频率
