### 1️⃣ 订单页面 (Orders)

#### 基础功能测试
```
□ 打开"订单"页面
□ 验证订单列表显示 (应显示 5 笔示例订单)
□ 验证状态卡片显示正确数量
  - 全部: 5
  - 待处理: 1
  - 处理中: 1
  - 已发货: 1
  - 已送达: 1
  - 已取消: 1
```

#### 搜索/过滤测试
```
□ 在搜索框输入订单号 "ORD-9921"
  - 应显示单条订单结果
□ 点击"待处理"状态卡片
  - 表格应只显示 1 笔订单
□ 清除过滤，验证列表恢复
```

#### CRUD 操作测试
```
□ 鼠标悬停订单行，出现"眼睛"和"垃圾桶"图标
□ 点击"眼睛"图标
  - 打开订单详情对话框
  - 显示订单号、客户、金额等信息
□ 点击对话框中的"编辑"按钮
  - 对话框转为编辑模式
  - 可修改状态（下拉菜单）
  - 修改状态为"已发货"并保存
  - 验证列表中状态已更新
□ 再次打开订单，点击"垃圾桶"图标
  - 订单应从列表中删除
```

#### AI 功能测试
```
□ 点击"自动处理订单"按钮
  - 显示加载动画
  - 待处理订单应变为处理中
□ 点击"智能通知客户"按钮
  - 显示操作完成消息
  - 提示已向 N 位客户发送通知
□ 其他 AI 按钮应显示相应的反馈
```

## 🐛 常见问题排查

### 问题 1: 编辑对话框不显示
**解决方案**:
```
□ 检查浏览器控制台是否有 JavaScript 错误
□ 刷新页面重试
□ 检查 Dialog 组件是否正确导入
```

### 问题 2: AI 功能无响应
**解决方案**:
```
□ 检查 Gemini API key 是否配置
□ 查看浏览器网络标签，API 请求是否成功
□ 检查 /api/gemini/generate 端点是否可达
```

### 问题 3: 数据修改后刷新页面消失
**预期行为** (当前正常):
```
✓ 这是预期的！数据存储在 React State 中
✓ 页面刷新会重置为初始数据
✓ 生产环境应接入真实数据库
```
